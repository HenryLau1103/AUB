AUB 是產品意圖與編碼 Agent 之間的開放合約層。建立語意化 Blueprint、對應正式元件，將同一份唯一真實來源交給 Codex、Claude Code、Copilot 或其他 Agent，再於 CI 逐項驗證 acceptance id。

黃金流程

從產品意圖到 Pull Request，只用一份合約

  1. 1

    組合或匯入

    建立單一畫面、匯入 Figma／Penpot bridge，或組成可導覽的多畫面專案。

  2. 2

    綁定正式元件

    將自訂類型對應到正式 module、export、source、Storybook 與 props。

  3. 3

    一次交付

    使用可攜式交付包或 MCP；不同 Agent 仍共享同一份核心合約。

  4. 4

    回報證據

    Agent 對應每個節點，並為每個 acceptance id 回報 pass、fail 或 needs-review。

  5. 5

    把關 PR

    AUB GitHub Action 會拒絕漂移、遺漏 mapping、驗收失敗與未解決事項。

為實作而設計

不是另一套設計畫布，也不是一次性程式碼生成器

設計工具最佳化創作，程式碼生成器最佳化速度；AUB 用 Git 與 CI 能理解的格式，跨工具、Agent、版本與審查保護產品意圖。

可版本化的唯一真實來源

JSON Schema、語意驗證、migration、diff 與 lock snapshot。

重用正式元件

將語意類型對應到既有 framework 元件，而不是重新製作相似品。

Agent-neutral 執行

Codex、Claude Code、Copilot 與通用 Agent 取得相同合約。

需要證據，不靠信心

Implementation report 與 PR check 會讓未完成工作清楚可見。

整合合約

讓 Agent 使用你已經上線的正式元件

自訂 registry 可指定正式 package、export symbol、source file、Storybook、文件，以及精確的 Blueprint-to-prop mapping。

  • 避免建立相似但不同的元件
  • 保留 repository 原生 token 與行為
  • 支援多種 framework 實作
{
  "name": "acme:insight_card",
  "isContainer": true,
  "implementations": [{
    "id": "react",
    "framework": "react",
    "module": "@acme/analytics-ui",
    "export": "InsightCard",
    "props": {
      "title": { "from": "content.title" }
    }
  }]
}

連接並驗證

需要可攜性時用檔案；Agent 需要即時存取時用 MCP。

匯出 .aub.zip，或讓支援 MCP 的 Agent 透過 stdio／HTTP 匯入、寫入、打包、解析、驗證、補全、diff、migrate、lock 與回報 Blueprint 和多畫面專案。再加入內建 GitHub Action，將合約變成 Pull Request 驗收閘門。

uses: HenryLau1103/AUB@main 驗證合約與實作證據
