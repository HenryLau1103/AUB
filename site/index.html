Golden path

One contract from intent to pull request

  1. 1

    Compose or import

    Build one screen, import a Figma/Penpot bridge, or compose a navigable multi-screen project.

  2. 2

    Bind real components

    Resolve custom types to production modules, exports, source files, Storybook, and props.

  3. 3

    Hand off once

    Use a portable package or MCP. The core contract stays identical across agents.

  4. 4

    Return evidence

    The agent maps every node and reports pass, fail, or needs-review for every acceptance id.

  5. 5

    Gate the PR

    The AUB GitHub Action rejects drift, missing mappings, failed criteria, and unresolved work.

Designed for implementation

Not another design canvas or one-shot code generator

Design tools optimize creation. Code generators optimize speed. AUB protects intent across tools, agents, revisions, and reviews with a format that Git and CI can understand.

Versioned source of truth

JSON Schema, semantic validation, migration, diff, and lock snapshots.

Production reuse

Map semantic types to existing framework components instead of recreating them.

Agent-neutral execution

Codex, Claude Code, Copilot, and generic agents receive the same contract.

Evidence, not confidence

Implementation reports and PR checks make incomplete work visible.

Integration contract

Point agents at the component you already ship

Custom registry entries can identify a production package, exported symbol, source file, Storybook story, documentation, and the exact Blueprint-to-prop mapping.

  • Prevent bespoke lookalike components
  • Keep repository-native tokens and behavior
  • Support multiple framework implementations
{
  "name": "acme:insight_card",
  "isContainer": true,
  "implementations": [{
    "id": "react",
    "framework": "react",
    "module": "@acme/analytics-ui",
    "export": "InsightCard",
    "props": {
      "title": { "from": "content.title" }
    }
  }]
}

Connect and verify

Files when you need portability. MCP when agents need live access.

Export an .aub.zip, or let MCP-capable agents over stdio or HTTP import, write, package, resolve, validate, scaffold, diff, migrate, lock, and report against Blueprints and multi-screen projects. Add the bundled GitHub Action to turn those contracts into a pull-request gate.
