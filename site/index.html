UI Blueprint Agent

Draw the interface.

AUB is a visual UI specification tool for people who need to communicate a screen precisely to Codex, Claude Code, GitHub Copilot, or another coding agent. Compose the screen, declare behavior and acceptance criteria, then export a structured package the agent can implement without guessing from prose or screenshots.

Open the live editor View on GitHub

The editor runs entirely in your browser. Nothing is uploaded.

How it works

  1. 1

    Compose visually

    Start from a common template or arrange registered semantic components on the canvas.

  2. 2

    Export a contract

    Capture hierarchy, layout, interactions, design tokens, responsive rules, and acceptance criteria.

  3. 3

    Hand off to an agent

    Give the package to a coding agent — over files or directly through the MCP server.

  4. 4

    Implement & verify

    The agent builds it, runs checks, and reports evidence for every acceptance id.

Why a contract beats a prompt

“Build a dashboard like Stripe” leaves critical decisions unstated. A screenshot shows appearance but not component intent, interaction outcomes, breakpoints, accessibility, or the acceptance bar. AUB turns those decisions into an explicit, testable contract.

Semantic components

Registered component types instead of anonymous rectangles.

Layout as a contract

Flex/grid and per-viewport geometry instead of inferred grouping.

Declared behavior

Explicit interactions, states, and responsive rules instead of guesses.

Enforceable acceptance

Testable acceptance ids spanning layout, interaction, responsive, and accessibility.

Connect your agent

Export an .aub.zip handoff package, or let MCP-capable agents call AUB tools directly over stdio: list_blueprints, get_blueprint, validate_blueprint, export_prompt, and submit_report.
