의도를 정의합니다.
프로덕션 컴포넌트를 재사용합니다.
증거로 PR을 검증합니다.

AUB는 제품 의도와 코딩 Agent 사이의 개방형 계약 계층입니다. 시맨틱 Blueprint를 만들고 프로덕션 컴포넌트에 연결한 뒤, 동일한 단일 기준을 Codex, Claude Code, Copilot 또는 다른 Agent에 전달하고 CI에서 모든 acceptance id를 검증합니다.

온라인 에디터 열기 GitHub 보기

Apache-2.0 · local-first · agent-neutral · 에디터는 데이터를 업로드하지 않습니다

62가지 시맨틱 컴포넌트 유형
18가지 반응형 시작 템플릿
16개의 Agent용 MCP 도구
4가지 인수 카테고리

골든 패스

제품 의도부터 Pull Request까지 하나의 계약으로

  1. 1

    구성 또는 가져오기

    단일 화면, Figma／Penpot bridge 또는 탐색 가능한 다중 화면 프로젝트를 만듭니다.

  2. 2

    프로덕션 컴포넌트 연결

    사용자 정의 유형을 프로덕션 module, export, source, Storybook, props에 연결합니다.

  3. 3

    한 번만 전달

    휴대 가능한 패키지나 MCP를 사용하며 모든 Agent가 동일한 핵심 계약을 공유합니다.

  4. 4

    증거 반환

    Agent가 모든 노드를 매핑하고 각 acceptance id를 pass, fail, needs-review로 보고합니다.

  5. 5

    PR 게이트

    AUB GitHub Action이 드리프트, 누락된 mapping, 인수 실패와 미해결 작업을 거부합니다.

구현을 위한 설계

또 하나의 디자인 캔버스나 일회성 코드 생성기가 아닙니다

디자인 도구는 제작을, 코드 생성기는 속도를 최적화합니다. AUB는 Git과 CI가 이해하는 형식으로 도구, Agent, 리비전과 리뷰 전반에서 의도를 보호합니다.

버전 관리 가능한 단일 기준

JSON Schema, 시맨틱 검증, migration, diff, lock snapshot.

프로덕션 자산 재사용

시맨틱 유형을 기존 framework 컴포넌트에 연결해 재구현을 방지합니다.

Agent-neutral 실행

Codex, Claude Code, Copilot과 범용 Agent가 동일한 계약을 받습니다.

신뢰가 아닌 증거

Implementation report와 PR check가 미완료 작업을 드러냅니다.

통합 계약

이미 배포 중인 프로덕션 컴포넌트를 Agent가 사용하게 합니다

사용자 정의 registry에 프로덕션 package, export symbol, source file, Storybook, 문서와 정확한 Blueprint-to-prop mapping을 지정할 수 있습니다.

  • 비슷하게만 생긴 별도 컴포넌트 방지
  • repository 고유 token과 동작 유지
  • 여러 framework 구현 지원
{
  "name": "acme:insight_card",
  "isContainer": true,
  "implementations": [{
    "id": "react",
    "framework": "react",
    "module": "@acme/analytics-ui",
    "export": "InsightCard",
    "props": {
      "title": { "from": "content.title" }
    }
  }]
}

연결 및 검증

이식성이 필요하면 파일을, Agent의 실시간 접근에는 MCP를 사용합니다.

.aub.zip을 내보내거나 MCP 지원 Agent가 stdio／HTTP를 통해 Blueprint와 다중 화면 프로젝트를 import, write, package, resolve, validate, scaffold, diff, migrate, lock, report하도록 합니다. 번들 GitHub Action을 추가하면 이 계약이 Pull Request 게이트가 됩니다.
